    Ripristino e ripristino dal backup Snapshot

    05/23/2023 Collaboratori

    "Precedente: Operazioni di backup basate su Snapshot di volumi globali non dati."

    La sezione seguente descrive i flussi di lavoro di ripristino e ripristino per un sistema single-tenant HANA MDC a host singolo. Vengono evidenziate le differenze per un sistema a più host. Gli esempi mostrano SAP HANA Studio come uno strumento per eseguire il ripristino manuale. Per le operazioni di recovery è inoltre possibile utilizzare istruzioni SAP HANA Cockpit o HANA SQL.

    Per ripristinare e ripristinare un sistema single-tenant SAP HANA MDC utilizzando SAP HANA Studio e Cloud Backup for Applications, attenersi alla seguente procedura:

    1. Preparare il processo di ripristino con SAP HANA Studio:

      1. Selezionare Recover System Database (Ripristina database di sistema) e confermare la chiusura del sistema SAP HANA.

      2. Selezionare il tipo di ripristino e la posizione di backup del registro.

        Viene visualizzato l’elenco dei backup dei dati.

      3. Selezionare il backup appropriato per visualizzare l’ID del backup esterno.

    2. Smontare i volumi di dati HANA.

    3. Eseguire il processo di ripristino con Cloud Backup per le applicazioni:

      1. Nella vista Backup, selezionare il backup che corrisponde all’ID di backup esterno o al campo dei commenti di SAP HANA Studio.

      2. Per avviare l’operazione di ripristino, selezionare Restore nel menu di scelta rapida.

    4. Montare i volumi di dati HANA.

    5. Eseguire il processo di ripristino del database di sistema con SAP HANA Studio:

      1. Fare clic su Refresh (Aggiorna) dall’elenco dei backup e selezionare il backup disponibile per il ripristino (indicato da un’icona verde).

      2. Avviare l’operazione di ripristino.

        Al termine dell’operazione di ripristino, viene avviato il database di sistema.

    6. Eseguire il processo di ripristino del database tenant con SAP HANA Studio:

      1. Selezionare Recover tenant Database (Ripristina database tenant) e selezionare il tenant da ripristinare.

      2. Selezionare il tipo di ripristino e la posizione di backup del registro.

    7. Viene visualizzato un elenco di backup dei dati. Poiché il volume di dati è già stato ripristinato, il backup del tenant indica come disponibile (in verde).

      1. Selezionare il backup e avviare il processo di ripristino. Al termine del processo di ripristino, il database del tenant viene avviato automaticamente.

    Viene visualizzato un elenco di backup dei dati. Poiché il volume di dati è già stato ripristinato, il backup del tenant indica come disponibile (in verde). Selezionare il backup e avviare il processo di ripristino. Al termine del processo di ripristino, il database del tenant viene avviato automaticamente.

    La seguente procedura descrive le operazioni di ripristino e ripristino del sistema HANA PR1.

    Preparare l’operazione di ripristino e ripristino

    1. Per avviare il ripristino del database di sistema in SAP HANA Studio, andare a Backup and Recovery > Recover System Database.

    2. Selezionare il tipo di ripristino e fare clic su Avanti.

    3. Fornire la posizione del catalogo di backup e fare clic su Avanti.

    4. Viene visualizzato un elenco dei backup disponibili in base al contenuto del catalogo di backup HANA. Scegliere il backup richiesto e annotare l’ID del backup esterno, ad esempio il backup più recente.

    5. Smontare tutti i volumi di dati.

      umount /hana/data/PR1/mnt00001
      Per un sistema host multiplo SAP HANA, è necessario smontare tutti i volumi di dati su ciascun host.

    Ripristinare il database HANA

    1. Per avviare l’operazione di ripristino, nella vista Backup, selezionare il backup, che viene visualizzato anche in HANA Studio, quindi selezionare Restore (Ripristina) nel menu di scelta rapida.

    2. Nella fase successiva, è possibile fornire script pre e post per l’operazione di ripristino.

      Il registro processi di Cloud Backup for Applications mostra lo stato di avanzamento dell’operazione di ripristino.

    3. Attendere il completamento del processo di ripristino e montare il volume di dati.

      mount /hana/data/SP1/mnt00001
      Per un sistema host multiplo SAP HANA, è necessario montare tutti i volumi di dati su ciascun host.

    Ripristinare il database di sistema

    1. Per aggiornare l’elenco dei backup disponibili, accedere a SAP HANA Studio e fare clic su Refresh (Aggiorna). Il backup ripristinato con Cloud Backup for Applications viene visualizzato con un’icona verde nell’elenco dei backup. Selezionare il backup e fare clic su Next (Avanti).

    2. Fornire la posizione dei backup del registro. Fare clic su Avanti.

    3. Selezionare le altre impostazioni desiderate. Assicurarsi che l’opzione Usa backup delta non sia selezionata. Fare clic su Avanti.

    4. Rivedere le impostazioni di ripristino e fare clic su fine.

    5. Viene avviato il processo di ripristino. Attendere il completamento del ripristino del database di sistema.

    Ripristinare il database tenant

    1. In SAP HANA Studio, selezionare la voce relativa al database di sistema e andare a Backup and Recovery > Recover tenant Database.

    2. Selezionare il tenant da ripristinare e fare clic su Avanti.

    3. Specificare il tipo di ripristino e fare clic su Next (Avanti).

    4. Confermare la posizione del catalogo di backup e fare clic su Avanti.

    5. Verificare che il database del tenant sia offline. Fare clic su OK per continuare.

      Poiché il ripristino del volume di dati si è verificato prima del ripristino del database di sistema, il backup del tenant è immediatamente disponibile.

    6. Selezionare il backup evidenziato in verde e fare clic su Avanti.

    7. Confermare la posizione di backup del registro e fare clic su Avanti.

    8. Selezionare le altre impostazioni desiderate. Assicurarsi che l’opzione Use Delta backups (Usa backup delta) non sia selezionata. Fare clic su Avanti.

    9. Esaminare le impostazioni di ripristino e avviare il processo di ripristino del database tenant facendo clic su fine.

    10. Attendere il completamento del ripristino e l’avvio del database tenant.

      Il sistema SAP HANA è operativo.

      Per un sistema SAP HANA MDC con più tenant, è necessario ripetere i passaggi da 16 a 25 per ciascun tenant.
